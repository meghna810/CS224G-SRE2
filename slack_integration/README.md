# Slack Integration for RootScout

Automatically send RCA analysis results to Slack when customer issues are detected.

## Quick Start

### 1. Get a Slack Webhook URL

1. Go to [Slack Apps](https://api.slack.com/apps)
2. Click "Create New App" → "From scratch"
3. Give it a name (e.g., "RootScout") and select your workspace
4. In the app settings, go to "Incoming Webhooks"
5. Toggle "Activate Incoming Webhooks" to ON
6. Click "Add New Webhook to Workspace"
7. Select the channel where you want notifications (e.g., #alerts, #incidents)
8. Copy the webhook URL (looks like: `https://hooks.slack.com/services/T00000000/B00000000/XXXXXXXXXXXXXXXXXXXX`)

### 2. Configure Your Environment

Add to your `.env` file:

```bash
# Slack Integration
SLACK_ENABLED=true
SLACK_WEBHOOK_URL=https://hooks.slack.com/services/YOUR/WEBHOOK/URL
```

### 3. Test the Integration

```bash
python slack_integration/test_slack.py
```

This will:
- Verify your webhook URL is configured correctly
- Send a test message to Slack
- Send a sample RCA notification

### 4. Use in Your Code

```python
from slack_integration import SlackNotifier

# Initialize (reads from .env automatically)
notifier = SlackNotifier()

# Send RCA analysis to Slack
analysis = {
    "root_cause_service": "cart-service",
    "confidence": 0.85,
    "reasoning": "Database timeout issue...",
    "recommended_action": "kubectl rollout restart deployment/cart-service"
}

notifier.send_rca_analysis(
    analysis=analysis,
    incident_title="Checkout Failures Detected",
    focus_service="cart-service",
    alert_severity="critical"  # or "warning", "info"
)
```

## Features

- **Rich Formatting**: Uses Slack Block Kit for beautiful, readable messages
- **Automatic Fallback**: If Slack is not configured, notifications are silently skipped
- **Severity Levels**: Support for critical 🔴, warning 🟡, and info 🔵 alerts
- **Confidence Indicators**: Visual indicators for high/medium/low confidence
- **Code Blocks**: Recommended actions shown in formatted code blocks

## Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `SLACK_ENABLED` | No | `true` | Enable/disable Slack notifications |
| `SLACK_WEBHOOK_URL` | Yes | None | Slack incoming webhook URL |

## Troubleshooting

### "Slack notifications disabled"
- Make sure `SLACK_ENABLED=true` in your `.env` file
- Or explicitly pass `enabled=True` when creating `SlackNotifier()`

### "No webhook URL configured"
- Make sure you've added `SLACK_WEBHOOK_URL` to your `.env` file
- Verify the webhook URL is valid and hasn't been revoked

### "Failed to send Slack notification: 403"
- Your webhook URL may have been revoked
- Go to Slack Apps and regenerate the webhook

### "Failed to send Slack notification: 404"
- The webhook URL is invalid or malformed
- Double-check you copied the entire URL

## API Reference

### SlackNotifier

#### `__init__(webhook_url=None, enabled=None)`
Initialize the Slack notifier.

- `webhook_url` (optional): Slack webhook URL. If not provided, reads from `SLACK_WEBHOOK_URL` env var.
- `enabled` (optional): Enable/disable notifications. If not provided, reads from `SLACK_ENABLED` env var.

#### `send_rca_analysis(analysis, incident_title, focus_service, alert_severity)`
Send RCA analysis results to Slack.

- `analysis` (dict): Analysis result from `RCAAgent.analyze()`
- `incident_title` (str): Title for the incident (default: "RCA Analysis Complete")
- `focus_service` (str, optional): Service that triggered the alert
- `alert_severity` (str): Severity level - "critical", "warning", or "info" (default: "warning")

Returns: `True` if successful, `False` otherwise

#### `send_simple_message(text)`
Send a simple text message to Slack.

- `text` (str): Message text

Returns: `True` if successful, `False` otherwise

#### `test_connection()`
Test the Slack webhook connection.

Returns: `True` if successful, `False` otherwise
