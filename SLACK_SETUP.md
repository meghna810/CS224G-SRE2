# Slack Integration Setup Guide

This guide will help you set up automatic Slack notifications for RCA analysis results.

## Prerequisites

- A Slack workspace where you have permission to add apps
- Access to your `.env` file

## Step-by-Step Setup

### 1. Create a Slack App and Get Webhook URL

1. **Go to Slack API Dashboard**
   - Visit: https://api.slack.com/apps
   - Click "Create New App"

2. **Choose "From scratch"**
   - App Name: `RootScout` (or any name you prefer)
   - Workspace: Select your workspace
   - Click "Create App"

3. **Enable Incoming Webhooks**
   - In the left sidebar, click "Incoming Webhooks"
   - Toggle "Activate Incoming Webhooks" to **ON**

4. **Add Webhook to Workspace**
   - Scroll down and click "Add New Webhook to Workspace"
   - Select the channel where you want notifications (e.g., `#alerts`, `#incidents`, `#sre-team`)
   - Click "Allow"

5. **Copy the Webhook URL**
   - You'll see a webhook URL that looks like:
     ```
     https://hooks.slack.com/services/T00000000/B00000000/XXXXXXXXXXXXXXXXXXXX
     ```
   - Copy this entire URL (you'll need it in the next step)

### 2. Configure Your Environment

1. **Open your `.env` file** (in the root of your project)

2. **Add or update these lines:**
   ```bash
   # Slack Integration
   SLACK_ENABLED=true
   SLACK_WEBHOOK_URL=https://hooks.slack.com/services/YOUR/WEBHOOK/URL
   ```

3. **Replace `YOUR/WEBHOOK/URL`** with the webhook URL you copied in Step 1

4. **Save the file**

### 3. Test Your Setup

Run the test script to verify everything works:

```bash
python slack_integration/test_slack.py
```

You should see:
- ✅ Connection test message in your Slack channel
- ✅ Sample RCA notification in your Slack channel

### 4. Run the Demo

Now when you run the demo, it will automatically send RCA results to Slack:

```bash
python demo.py
```

After the RCA analysis completes, check your Slack channel for a formatted notification with:
- Root cause service
- Confidence level
- Analysis reasoning
- Recommended action

## Troubleshooting

### I don't see the webhook URL after creating the app

- Make sure you **toggled "Activate Incoming Webhooks" to ON** first
- Then click "Add New Webhook to Workspace"
- The URL will appear in a table at the bottom of the page

### The test script says "Slack notifications disabled"

- Check that `SLACK_ENABLED=true` in your `.env` file (not `false`)
- Make sure there are no typos

### The test script says "webhook URL not configured"

- Verify you added `SLACK_WEBHOOK_URL=...` to your `.env` file
- Make sure the entire webhook URL is copied (should start with `https://hooks.slack.com/services/`)
- Remove any extra spaces or quotes around the URL

### Messages aren't appearing in Slack

- Verify the webhook URL is correct
- Check that the app has permission to post to the selected channel
- The webhook may have been revoked - go back to Slack Apps and regenerate it

### I want to use a different channel

- Go back to https://api.slack.com/apps
- Select your RootScout app
- Go to "Incoming Webhooks"
- Click "Add New Webhook to Workspace"
- Select the new channel
- Copy the new webhook URL to your `.env` file

## Disabling Slack Notifications

To temporarily disable Slack notifications without removing your webhook URL:

```bash
# In your .env file
SLACK_ENABLED=false
```

To re-enable:
```bash
SLACK_ENABLED=true
```

## Security Best Practices

1. **Never commit your `.env` file** to git (it should be in `.gitignore`)
2. **Keep your webhook URL secret** - anyone with it can post to your channel
3. **Revoke and regenerate** the webhook if it's exposed
4. **Use different webhooks** for different environments (dev, staging, prod)
